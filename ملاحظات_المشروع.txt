2. نظام المخزون الثنائي الطبقة (كرتون + وحدة)
--------------------------------------------------
- باركودان منفصلان لكل منتج:
  * باركود الوحدة (للبيع بالتجزئة)
  * باركود الكرتون (للشراء بالجملة)
- مستويا مخزون مستقلان:
  * رف المتجر (shelfStock): بالوحدات
  * المستودع (warehouseStock): بالكراتين
- وحدات الكرتون: يحدد الأدمن عدد الوحدات في الكرتون
- نقل ذكي: نقل كرتون/كراتين من المستودع للرف مباشرة
- تنبيهات منخفض المخزون مستقلة لكل مستوى

3. محرك التسعير الديناميكي
-----------------------------
- هامش ربح مُدار لكل منتج (افتراضي: 15%)
- عند تغيير سعر الجملة للكرتون:
  * يُحسب سعر الجملة للوحدة = سعر الكرتون / عدد الوحدات
  * يُحسب سعر التجزئة = سعر الجملة × (1 + الهامش%)
  * التحديث تلقائي في الوقت الفعلي عند الإدخال
- لا تغيير يدوي للأسعار دون مراجعة الهامش

4. المحرك المالي والتسوية النقدية
-------------------------------------
معادلة الربح الصافي:
  صافي الربح = (إيرادات التجزئة - تكلفة الجملة) - (مصاريف يومية + تسبقات الموظفين + خصومات التأخير)

نظام الورديات (الفكة - Cash Reconciliation):
- القابض يمسح باركود بطاقته لفتح الوردية
- إدخال المبلغ الافتتاحي (starting float)
- عند إغلاق الوردية:
  * إدخال النقد الفعلي المعدود
  * النظام يحسب: المتوقع = الافتتاحي + مبيعات الكاش
  * يُسجل الفرق: عجز (أحمر) أو فائض (أخضر)
  * مرتبط باسم القابض تلقائياً

5. إدارة المصاريف والتوزيع اليومي
---------------------------------------
- يسجل الأدمن المصاريف الشهرية (إيجار/ضرائب/كهرباء/ماء/أخرى)
- النظام يقسم المصروف الشهري على أيام الشهر
- يُعرض المصروف اليومي في لوحة الربح الصافي
- أنواع المصاريف: شهري / يومي / لمرة واحدة

6. التسبقات والخصومات
-------------------------
- سجل التسبقات (تسبق): للموظف مع السبب والشهر
- سجل خصومات التأخير
- خصم تلقائي من الراتب الشهري
- يُعرض في تقرير الربح الصافي

7. نظام الحضور بالباركود
--------------------------
- كل موظف له باركود فريد للحضور:
  * EMP-0001: أيمن المدير
  * EMP-0002: كريم القابض
  * EMP-0003: سامي المشري
  * EMP-0004: علي العامل
- الوردية لا تُفتح إلا بعد مسح الباركود بنجاح
- تُسجل الوردية باسم الموظف تلقائياً

8. نظام المهام وطلبات النقل
-------------------------------
- إنشاء مهمة لعامل (نقل، تقرير، تلف، أخرى)
- نقاط نشاط لكل مهمة منجزة
- قائمة موافقة الأدمن:
  * العامل يُنجز المهمة → تنتقل لـ "مكتملة"
  * الأدمن يوافق → تُضاف النقاط للراتب
  * الأدمن يرفض → لا نقاط
- نقاط شهرية تراكمية لكل موظف

مساعد الذكاء الاصطناعي الذكي:
- عندما يسأل القابض عن منتج المخزون منخفض في الرف لكن المستودع به بضاعة:
  → الذكاء الاصطناعي يُنشئ مهمة نقل تلقائية للعامل
- مثال: "هل يوجد حليب؟" → إذا الرف منخفض: ينشئ مهمة نقل كرتون من المستودع

9. لوحة البيع المتقدمة للقابض (POS Terminal)
---------------------------------------------
- مسح باركود الوردية لفتح الجلسة (إلزامي)
- بحث بالاسم أو الباركود (وحدة أو كرتون)
- لوحتان: يسار للمنتجات، يمين للسلة
- إدخال الوزن للمنتجات بالكيلو
- إيقاف البيع مؤقتاً (Hold Cart) وحفظه
- خصومات (مبلغ ثابت أو نسبة)
- طرق الدفع: نقداً / كرني / بطاقة
- إغلاق الوردية مع تسوية نقدية
- طباعة الفاتورة العربية (دعم 80مم)
- مساعد الذكاء الاصطناعي داخل الشاشة

10. لوحات التحكم (حسب الدور)
-------------------------------
للأدمن (لوحة كاملة):
  ✓ إيرادات اليوم والشهر
  ✓ الربح الصافي اليومي (بعد خصم المصاريف والتسبقات)
  ✓ الربح الإجمالي الشهري
  ✓ مصاريف الشهر
  ✓ تسبقات الموظفين
  ✓ رسم بياني للمبيعات (7 أيام) مع منحنى الربح الصافي
  ✓ أكثر المنتجات مبيعاً
  ✓ المنتجات قريبة الانتهاء
  ✓ إجمالي الديون (كرني)
  ✓ التقارير المعلقة

11. إدارة الموظفين والرواتب (للأدمن فقط)
-------------------------------------------
- إضافة/تعديل/حذف الموظفين مع باركودات الحضور
- سجلات الرواتب الشهرية: أساسي + مكافأة - خصم = صافي
- تسبقات وخصومات مُدمجة في الراتب
- نقاط النشاط تُضاف كمكافأة بعد موافقة الأدمن

ثالثاً: واجهات برمجة التطبيقات الجديدة (API)
====================================================================

الورديات:
  POST /api/shifts/open          → فتح وردية بمسح الباركود
  POST /api/shifts/close         → إغلاق وردية مع التسوية النقدية
  GET  /api/shifts               → قائمة كل الورديات
  GET  /api/shifts/active/:id    → الوردية المفتوحة لقابض معين

المهام:
  GET    /api/tasks              → قائمة المهام (بفلتر الحالة/النوع)
  POST   /api/tasks              → إنشاء مهمة جديدة
  PATCH  /api/tasks/:id/complete → إنجاز المهمة (من العامل)
  PATCH  /api/tasks/:id/approve  → موافقة/رفض المهمة + منح النقاط (أدمن)
  DELETE /api/tasks/:id          → حذف مهمة

المصاريف:
  GET    /api/expenses           → قائمة المصاريف (بفلتر الشهر)
  GET    /api/expenses/daily-total → إجمالي المصروف اليومي
  POST   /api/expenses           → إضافة مصروف (مع حساب التوزيع اليومي)
  DELETE /api/expenses/:id       → حذف مصروف

التسبقات والخصومات:
  GET    /api/advances           → قائمة التسبقات/الخصومات
  GET    /api/advances/summary/:month → ملخص شهري لكل موظف
  POST   /api/advances           → إضافة تسبقة أو خصم
  PATCH  /api/advances/:id/mark-deducted → تأكيد الخصم من الراتب
  DELETE /api/advances/:id       → حذف

المنتجات (محدّث):
  GET  /api/products/barcode/:barcode → بحث بالباركود (وحدة أو كرتون)
  POST /api/products/:id/restock → نقل كراتين من المستودع للرف

لوحة التحكم (محدّث):
  GET /api/dashboard/summary     → + صافي ربح يومي، مصاريف، تسبقات
  GET /api/dashboard/net-profit  → تفصيل الربح الصافي الشهري
  GET /api/dashboard/sales-chart → + منحنى صافي الربح

الذكاء الاصطناعي (محدّث):
  POST /api/ai/inventory-query   → + إنشاء مهمة نقل تلقائي (createTaskIfNeeded)
